New · Phase 1

What we added: Attention Gating

The core insight: not all particles are equally relevant for predicting a masked particle. A b-quark is more informative for predicting another b-quark than a soft wide-angle emission. We add a learned scalar weight per particle — computed from the Lorentz-invariant pairwise interaction matrix U — applied before the context encoder sees the sequence.

Where does U come from?

U is computed by ParticleProcessor

For every pair of particles (i, j) in the jet, the processor computes four Lorentz-invariant geometric quantities from kt-clustering variables. These encode the structure of jet splittings — the same variables used on the Lund jet plane in particle physics.

U shape: (B, N, N, 4)

A 4-vector for every particle pair. Diagonal = 0 (self-interaction). Padding pairs = −10⁹ (numerically excluded).

How the gate trains: No special loss — the gate's MLP parameters (4×16 + 16×16 + 16×1 = 336 weights) receive gradients from the JEPA embedding loss via the context encoder. When a particle's gate is wrong, the context encoder produces a bad prediction, the loss is high, and backprop adjusts the gate.

What's next

🧪

Run Phase 1 experiments

Compare JEPA+gate vs JEPA (no gate) across 3 seeds. Hypothesis: gate improves representation quality for structurally complex decays (t→bqq, H→4q).
🎭

Multi-particle masking (K=1,2,4)

Currently mask 1 particle at a time. Masking K>1 simultaneously forces the predictor to reason about multiple missing particles jointly — harder task, potentially richer representations.
📊

Phase 2: Ablation sweep

Predictor depth (2/4/6), bottleneck dim (32/64/128), EMA schedule, loss variant (MSE / cosine / smooth-L1).
⚡

Phase 3: Triton kernel for EquiLinear

Profile forward+backward, locate bottleneck in L-GATr's 16-dim multivector contraction, implement fused Triton kernel to reduce memory traffic on A100.
